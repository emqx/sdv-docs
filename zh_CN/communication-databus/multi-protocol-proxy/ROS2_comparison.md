# ROS2 自带 MQTT 客户端和 ROS2DDS 桥接工具对比

## Most popular ways

1. SDV-Flow DDS Proxy
2. MQTT client in ROS2

## 对比

+ 延时

  - SDV-Flow DDS Proxy

  `La = Ave(Tx - Ty)`

  `La` is the average duration from ROS2 to MQTT. `Tx` is the start time of a transmission while  `Ty` is the end time of the transmission.

  - MQTT client in ROS2

  Here we use the echo way mentioned in [readme](https://github.com/ika-rwth-aachen/mqtt_client#latency-computation).

  `Lb = Ave(Ts - Tz) / 2`

  `Lb` is the average duration from ROS2 to MQTT. `Ts` is the start time of a transmission from a ROS2 domain while  `Ty` is the end time of the transmission at another ROS2 domain.

  | Project                              | SDV-Flow DDS Proxy | MQTT client in ROS2 |
  | ------------------------------------ | ---------------- | ------------------- |
  | Average Latency ROS2->MQTT (10msg/s) | 98ms             | 94ms                |
  | Average Latency MQTT->ROS2 (10msg/s) | 43ms             | 94ms                |
